# プログラムアップデートに向けたペルソナチーム

今後のプログラムアップデートに関する議論を効果的に進めるため、以下の16名のペルソナで構成されるチームを定義する。

このチームは、プログラムに関わる主要な組織（**学生、企業、大学、商工会議所**）の代表者で構成され、各組織内に存在する立場や関心事の違い（健全な対立構造）を明確にすることで、利害調整や課題の発見を促進することを目的とする。

---

## 1. 学生ペルソナ（4パターン）

夜間部学生の多様な背景・動機を反映。

### 1-1. 経済自立型（学生A1）

| 項目 | 内容 |
|------|------|
| **学年** | 工学部2年生 |
| **経済状況** | 厳しい（学費・生活費を自弁） |
| **成績** | 中程度 |
| **技術への関心** | 高い |
| **特徴** | 働きながら学ぶことが前提。収入の安定が最優先だが、将来につながる経験も積みたい。 |

**関心事:**
- 「時給はいくらか？安定して稼げるか？」
- 「シフトの融通は利くか？試験期間は休めるか？」
- 「交通費は出るか？勤務地は通いやすいか？」
- 「将来の就職に少しでも役立つ経験になるか？」

---

### 1-2. キャリア志向型（学生A2）

| 項目 | 内容 |
|------|------|
| **学年** | 工学部3年生 |
| **経済状況** | 普通 |
| **成績** | 優秀 |
| **技術への関心** | 非常に高い |
| **特徴** | 実務経験を通じてスキルアップし、就職活動でアピールしたい。意欲的で自己成長に貪欲。 |

**関心事:**
- 「最新の技術や設備に触れられるか？」
- 「実務経験証明書に書ける具体的な成果を出せるか？」
- 「将来の就職先として検討できる企業か？」
- 「エンジニアとして成長できる環境か？」

---

### 1-3. 学業優先型（学生A3）

| 項目 | 内容 |
|------|------|
| **学年** | 工学部3年生 |
| **経済状況** | 余裕あり（家庭からの支援） |
| **成績** | 優秀 |
| **技術への関心** | 中程度 |
| **特徴** | 単位取得・卒業研究が最優先。アルバイトは「経験として少しやってみたい」程度。 |

**関心事:**
- 「週に何時間くらいの負担か？」
- 「学業に支障が出ないか？」
- 「研究室の活動と両立できるか？」
- 「面倒な手続きや拘束は少ないか？」

---

### 1-4. 模索中型（学生A4）

| 項目 | 内容 |
|------|------|
| **学年** | 工学部2年生 |
| **経済状況** | 普通 |
| **成績** | やや低め |
| **技術への関心** | 低〜中 |
| **特徴** | 工学部に入ったものの、将来の方向性が定まっていない。「とりあえず」で参加を検討。 |

**関心事:**
- 「自分に向いている仕事が見つかるか？」
- 「難しい仕事を任されないか？ついていけるか？」
- 「怒られたり、厳しく指導されないか？」
- 「合わなかったらすぐ辞められるか？」

---

## 2. 企業ペルソナ：経営者（3パターン）

企業の採用・教育に対する温度差を反映。

### 2-1. 積極投資型（経営者B1）

| 項目 | 内容 |
|------|------|
| **役職** | 中小製造業・代表取締役 |
| **従業員規模** | 30〜50名 |
| **採用意欲** | 高い |
| **教育投資** | 惜しまない |
| **特徴** | 若手育成を経営戦略の柱と捉える。地域貢献にも積極的。 |

**関心事:**
- 「優秀な学生を早期に囲い込みたい」
- 「卒業後の正社員採用につなげたい」
- 「地域の大学との連携実績を作りたい」
- 「社内の活性化につなげたい」

---

### 2-2. 様子見型（経営者B2）

| 項目 | 内容 |
|------|------|
| **役職** | 中小製造業・代表取締役 |
| **従業員規模** | 10〜20名 |
| **採用意欲** | 中程度 |
| **教育投資** | 慎重 |
| **特徴** | 人手不足で興味はあるが、学生採用のリスクや手間を懸念。 |

**関心事:**
- 「本当に戦力になるのか？」
- 「トラブルが起きたら誰が責任を取るのか？」
- 「手続きや管理の手間はどれくらいか？」
- 「他社の成功事例を見てから判断したい」

---

### 2-3. 即戦力志向型（経営者B3）

| 項目 | 内容 |
|------|------|
| **役職** | 中小製造業・代表取締役 |
| **従業員規模** | 5〜10名 |
| **採用意欲** | 条件付き |
| **教育投資** | 最小限 |
| **特徴** | 教育に時間をかける余裕がない。即戦力か、少なくとも指示待ちでない人材を求める。 |

**関心事:**
- 「教えなくても動ける学生はいるか？」
- 「短期間で辞められると困る」
- 「最低限のスキル（PC操作、工具の使い方等）はあるか？」
- 「費用対効果が見合うか？」

---

## 3. 企業ペルソナ：現場担当（3パターン）

学生を直接指導する立場の温度差を反映。

### 3-1. 育成熱心型（現場担当C1）

| 項目 | 内容 |
|------|------|
| **役職** | 製造部門・主任クラス |
| **年代** | 30代 |
| **指導意欲** | 高い |
| **時間的余裕** | ある程度確保 |
| **特徴** | 後輩育成にやりがいを感じる。自分の知識・技術を伝えたい。 |

**関心事:**
- 「素直に学ぶ姿勢があるか？」
- 「長期的に関われる学生か？」
- 「自分の仕事の意義を伝えられる機会になる」
- 「将来の正社員候補として育てたい」

---

### 3-2. 多忙型（現場担当C2）

| 項目 | 内容 |
|------|------|
| **役職** | 製造部門・リーダークラス |
| **年代** | 40代 |
| **指導意欲** | あるが… |
| **時間的余裕** | ほぼない |
| **特徴** | 教えたい気持ちはあるが、自分の業務で手一杯。 |

**関心事:**
- 「指導に割く時間が本当にあるか不安」
- 「最低限の報連相ができる学生か？」
- 「手がかからない学生ならありがたい」
- 「自分の仕事を手伝ってもらえるなら助かる」

---

### 3-3. 懐疑型（現場担当C3）

| 項目 | 内容 |
|------|------|
| **役職** | 製造部門・ベテラン技術者 |
| **年代** | 50代 |
| **指導意欲** | 低い |
| **時間的余裕** | — |
| **特徴** | 過去に学生アルバイトで苦い経験あり。期待していない。 |

**関心事:**
- 「どうせすぐ辞めるんでしょ？」
- 「仕事を教えても無駄になる」
- 「安全面が心配。怪我でもされたら困る」
- 「余計な仕事を増やさないでほしい」

---

## 4. 大学ペルソナ（4パターン）

大学内部の異なる優先事項（効率 vs 学生保護）とITリテラシー（得意 vs 苦手）を反映。

| | IT得意 | IT苦手 |
|---|--------|--------|
| **効率優先** | D1: デジタル推進型 | D2: アナログ効率型 |
| **学生保護優先** | D3: データ活用型 | D4: 対面重視型 |

### 4-1. デジタル推進型（大学担当D1）

| 項目 | 内容 |
|------|------|
| **所属** | 総務部 地域連携担当 |
| **役割** | プログラムの日常運用 |
| **優先事項** | 業務効率・省力化 |
| **ITリテラシー** | 高い |
| **年代** | 30〜40代 |
| **特徴** | 新しいツールやシステムを積極的に導入。自動化・効率化を推進。 |

**関心事:**
- 「Googleフォームやスプレッドシートで自動化したい」
- 「問い合わせはFAQやチャットボットで対応したい」
- 「紙の書類や電話対応は極力なくしたい」
- 「データで進捗や成果を可視化したい」

---

### 4-2. アナログ効率型（大学担当D2）

| 項目 | 内容 |
|------|------|
| **所属** | 総務部 地域連携担当 |
| **役割** | プログラムの日常運用 |
| **優先事項** | 業務効率・省力化 |
| **ITリテラシー** | 低い |
| **年代** | 50代 |
| **特徴** | 既存の紙・電話ベースの運用を好む。新システム導入に抵抗感。 |

**関心事:**
- 「今までのやり方で回っているのに、なぜ変える必要があるのか」
- 「新しいシステムを覚える時間がない」
- 「パソコンが苦手な企業担当者もいる。電話の方が確実」
- 「システムトラブルが起きたら誰が対応するのか」

---

### 4-3. データ活用型（大学担当D3）

| 項目 | 内容 |
|------|------|
| **所属** | 学生厚生課 |
| **役割** | 学生の安全・権利保護 |
| **優先事項** | リスク管理・学生支援 |
| **ITリテラシー** | 高い |
| **年代** | 30〜40代 |
| **特徴** | データ分析で学生のリスクを早期発見したい。エビデンスベースの支援を志向。 |

**関心事:**
- 「勤務時間や出勤状況をデータで監視し、異常を検知したい」
- 「トラブル報告のパターンを分析して予防策を立てたい」
- 「学生アンケートを定期的に実施し、満足度を数値化したい」
- 「問題企業を客観的なデータで特定したい」

---

### 4-4. 対面重視型（大学担当D4）

| 項目 | 内容 |
|------|------|
| **所属** | 学生厚生課 |
| **役割** | 学生の安全・権利保護 |
| **優先事項** | リスク管理・学生支援 |
| **ITリテラシー** | 低い |
| **年代** | 50代 |
| **特徴** | 学生との直接対話を重視。システム任せでは本当の問題が見えないと考える。 |

**関心事:**
- 「学生の顔を見て話さないと、本当の悩みはわからない」
- 「データだけでは見落とす問題がある」
- 「困った時に相談できる窓口が必要」
- 「システムに頼りすぎると、人間関係が希薄になる」

---

## 5. 商工会議所ペルソナ（2パターン）

地域の産業支援組織としての立場を反映。

### 5-1. 地域活性推進型（商工会議所E1）

| 項目 | 内容 |
|------|------|
| **役職** | 経営支援課・課長 |
| **スタンス** | 積極的 |
| **特徴** | 産学連携で地域経済を活性化したい。新しい取り組みに前向き。 |

**関心事:**
- 「地域の中小企業と大学をつなぐ良い機会」
- 「成功事例を作って他の企業にも広げたい」
- 「商工会議所の存在価値をアピールできる」
- 「地域の若者流出を食い止めたい」

---

### 5-2. 会員サービス型（商工会議所E2）

| 項目 | 内容 |
|------|------|
| **役職** | 会員サービス課・担当者 |
| **スタンス** | 中立的 |
| **特徴** | 会員企業へのサービス提供が主眼。プログラムは「選択肢の一つ」として紹介。 |

**関心事:**
- 「会員企業にとってメリットがあるか？」
- 「トラブルが起きた時に商工会議所の責任にならないか？」
- 「紹介するなら、しっかりした制度であってほしい」
- 「問い合わせ対応の手間は増やしたくない」

## ペルソナ間の対立構造

このペルソナ構成は、プロジェクトに内在する**「健全な対立構造」**を明確に反映しており、課題発見に有効である。

### 学生間の対立
- `経済自立型A1（収入重視）` vs `キャリア志向型A2（成長重視）`
- → 時給・条件と、経験・スキルのバランスをどう設計するか

### 企業内部の対立
- `積極投資型B1（育成したい）` vs `即戦力志向型B3（教える余裕がない）`
- `育成熱心型C1（教えたい）` vs `懐疑型C3（期待していない）`
- → 大学の事前研修がどこまで現場の負担を軽減できるか

### 大学内部の対立
- `デジタル推進型D1（効率×IT得意）` vs `アナログ効率型D2（効率×IT苦手）`
- → システム化の推進度合い、導入スピード
- `データ活用型D3（保護×IT得意）` vs `対面重視型D4（保護×IT苦手）`
- → データ監視 vs 対話による学生支援のアプローチ
- `D1・D3（IT得意）` vs `D2・D4（IT苦手）`
- → 新システム導入時の合意形成の難しさ

### 組織間の対立
- `学生A1〜A4（良い条件で働きたい）` vs `経営者B3（コストを抑えたい）`
- `商工会議所E1（積極推進）` vs `商工会議所E2（慎重姿勢）`
- → プログラムが各ステークホルダーにとって「Win-Win」になっているか

## 補足

この16名のペルソナに加え、プロジェクト全体の方向性を決定し、各ペルソナ間の利害対立を最終的に仲裁する**「プロジェクトオーナー（工学部第二部長）」**の視点も、議論の際には暗黙的に保持する。
