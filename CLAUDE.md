# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## プロジェクト概要

工学部第二部が主導する「地域共育プログラム」のドキュメントリポジトリ。夜間部学生と地域中小企業を接続するプラットフォーム型（掲示板方式）の産学連携モデルを構築するプロジェクト。

**コア・コンセプト**: 大学が「仲介」ではなく「信頼できる情報の場」と「学生の質保証（認定）」のみを提供する。

## ファイル構成

ファイル名先頭の数字で対象読者・カテゴリを分類：
- `00_`: プロジェクト全体定義・Web要件（最重要は`00_プロジェクト定義書.md`）
- `10_`: 管理者向け内部規定・マニュアル
- `30_`: 学生向け資料（研修・規約・テンプレート）
- `40_`: 企業向け資料（案内・規約）

## 編集時の注意

- Obsidian形式のMarkdownを使用（`[表示名](ファイル名.md)`形式の内部リンク）
- コールアウト構文を使用: `> [!NOTE]`, `> [!WARNING]`, `> [!DANGER]`, `> [!ABSTRACT]`

## メタデータ仕様

各ドキュメントには以下のYAMLフロントマターを必ず含めること：

```yaml
---
title: ドキュメントタイトル
category: 00_全体・要件定義 | 10_内部規定・マニュアル | 30_学生向け資料 | 40_企業向け資料
doc_type: 定義書 | 設計書 | 規約 | マニュアル | 研修資料 | 雛形 | 案内
audience:
  - 学生
  - 企業
  - 事務局
version: "1.0"
status: 🔴Draft | 🟡Review | 🟢Release
created: YYYY-MM-DD
updated: YYYY-MM-DD
owner: 工学部第二部長
review_cycle: 年1回 | 学期毎 | 随時
---
```

### フィールド説明
| フィールド | 必須 | 説明 |
|------------|------|------|
| `title` | ✅ | ドキュメントのタイトル |
| `category` | ✅ | ファイル番号に対応するカテゴリ |
| `doc_type` | ✅ | 文書の種類 |
| `audience` | ✅ | 対象読者（複数可） |
| `version` | ✅ | セマンティックバージョン（文字列） |
| `status` | ✅ | 承認状態 |
| `created` | ✅ | 作成日 |
| `updated` | ✅ | 最終更新日（編集時に必ず更新） |
| `owner` | ✅ | 承認権限者（常に「工学部第二部長」） |
| `review_cycle` | ✅ | 定期見直し頻度 |

## 公開リポジトリのため厳守事項

このリポジトリは**GitHub上で公開**されています。以下の情報は**絶対に含めないでください**：

1. **個人情報**: 実在する個人名、メールアドレス、電話番号、住所
2. **特定企業名**: 東京電機大学工学部第二部以外の実在する企業名・団体名
3. **機密情報**: 内部の連絡先、非公開の運用情報

### 許容される表現
- プレースホルダー: `〇〇株式会社`、`example@dendai.ac.jp`、`03-XXXX-XXXX`
- 匿名化されたペルソナ: `学生A`、`経営者B`、`事務担当C` など
- 役職のみの記載: `工学部第二部長`（個人名なし）

## プロジェクトのキーポイント

1. **Pull型限定**: 学生から企業への応募のみ（逆オファー禁止）
2. **契約期間**: 学期末（3月31日/9月30日）区切りの更新推奨
3. **認定方式**: デジタル認定証（合格メール）、窓口業務ゼロ化
4. **保険**: 労災保険は企業の義務（大学の学研災は適用外）
5. **トリアージ**: Lv.1（当事者解決）→ Lv.2（認定停止）→ Lv.3（永久排除）の3段階
